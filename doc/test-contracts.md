# Tactical Test Source Contracts

This document holds the detailed, per-test source-contract catalog for the
tactical test suites under `tests/tactical/*` (and their GUI-live counterparts in
`tests/gui/*`). It was extracted out of `AGENTS.md` so the auto-loaded agent
entry-point context stays slim; consult this file only when you are adding to or
modifying these specific tactical tests.

These are the key source contracts locked by the tactical tests
(`FTacticalGameMechanicsTest`, `FTacticalCombatReportTest`,
`FTacticalAttackResult`, `FTacticalGameMockUITest`, `WXTacticalUIAdapterTest`,
`FTacticalMineDamageFlowTest`, `FTacticalSeekerMovementTest`,
`FTacticalBattleBoardRendererDelegationTest`, and related fixtures):

- **SMC-04**: `drawPlaceMines()` initializes its source-list `y` from `getActionButtonRowBottom()`, not `BORDER` (subsequently changed by SMRIV-01 to `getActionPromptLineY(0)` for the right-column layout); `ACTION_PROMPT_*` constants reserve prompt-text height before the action-button row across all tactical phases.
- **SMC-05**: `drawSeekerMissiles()` contains a `PH_ATTACK_FIRE` branch sourcing pending hexes via `getAllPendingOffensiveFireSeekerHexes()`; pending hexes drawn without rotation, active seekers with heading rotation, branch exits early.
