Implementation context:
- Root cause fixed: drawMoveShip() (src/tactical/FBattleDisplay.cpp) computed `instrW` from dc.GetTextExtent() on the full unwrapped prompt strings (previously :1152-1156), even though the same strings are always rendered wrapped via drawWrappedActionPrompt(). Added a new helper `int FBattleDisplay::measureWrappedActionPromptWidth(wxDC&, const wxString&, int maxWidth) const` (declared in include/tactical/FBattleDisplay.h, defined in src/tactical/FBattleDisplay.cpp right after drawWrappedActionPrompt()) that runs the same greedy word-wrap algorithm as countWrappedActionPromptLines()/drawWrappedActionPrompt() but returns the widest individual wrapped line's pixel width. drawMoveShip() now computes `instrW` as the max of measureWrappedActionPromptWidth() over turnPrompt, detailPromptOne, detailPromptTwo, and detailPromptThree (all measured against promptMaxWidth), instead of dc.GetTextExtent() on the raw strings.
- Text wrap: buildMovePromptText() (src/tactical/FBattleDisplay.cpp, declared in include/tactical/FBattleDisplay.h) gained a 4th output parameter `wxString & detailPromptThree`. The former single long line "Press the 'Movement Done' button when all ships have been assigned their movement instructions." is now split into detailPromptTwo = "Press the 'Movement Done' button when all ships" and detailPromptThree = "have been assigned their movement instructions." in the normal-movement branch; detailPromptThree is cleared to empty in the other two branches (stopped-facing-selection and no-ship-selected). All four prompt strings are drawn via drawWrappedActionPrompt() and counted via countWrappedActionPromptLines() in both refreshMovePromptReservation() and drawMoveShip().
- Deduplication: drawMoveShip() previously duplicated the prompt-construction logic already present in buildMovePromptText() (used by refreshMovePromptReservation()). drawMoveShip() now calls buildMovePromptText() directly instead of re-deriving turnPrompt/detailPromptOne/detailPromptTwo inline, so the two call sites can never diverge on prompt text or line count.
- Caption: drawMoveShip() draws a 4-line caption directly above the Turn Left/Turn Right button panel using a smaller 8pt font (wxFONTFAMILY_SWISS, normal): 'If at the end of a ship's movement' / 'you want to make a facing change' / '(and have remaining MR), use these' / 'buttons to make a single final turn.' -- exact wording as specified in the task, hardcoded into 4 lines (each ~33-37 characters) rather than dynamically wrapped, so the line count and per-line width are deterministic. Caption width (captionW) is measured via dc.GetTextExtent() per line; blockWidth = max(m_turnButtonPanel->GetBestSize().GetWidth(), captionW) is used for the fit check against the ship-stats column, and the caption's top-left is placed at (panelX, getActionPromptLineY(0)); the button panel itself is placed captionH + BORDER below that.
- Panel growth: before the fit check, drawMoveShip() computes blockBottom = panelY + m_turnButtonPanel->GetBestSize().GetHeight() + BORDER (where panelY already accounts for the caption height) and, if that exceeds m_lowerPanelLayoutState.requestedDisplayHeight, sets requestedDisplayHeight = blockBottom and calls applyRequestedDisplayHeight() so the lower panel grows to fit caption + buttons without clipping, mirroring the existing growth pattern used elsewhere in this file (drawPlaceMines/drawPlaceSeekers/drawSeekerActivation/drawCurrentShipStats stacked mode).
- Fit/visibility guard unchanged in spirit: the panel (and now the caption) are only drawn/shown when panelX + blockWidth + 2*BORDER <= statsLeft (statsLeft = m_lowerPanelLayoutState.shipStatsLeftMargin when initialized). Because instrW is now bounded by actual wrapped line widths (well under the ~273px promptMaxWidth column, since each of the four lines is now short), lMargin no longer overshoots statsLeft=300 at the default window size, so this guard now passes and the panel (with caption) is shown instead of hidden on every paint.
- Untouched: the Movement Done button logic, onMoveDone()/onTurnLeft()/onTurnRight() handlers, canApplyEndOfMoveTurnLeft()/canApplyEndOfMoveTurnRight() enable-state wiring, drawCurrentShipStats(), and all other draw*() phase methods (drawPlaceMines, drawPlaceSeekers, drawSeekerActivation, drawDefensiveFire, drawAttackFire, etc.) and their instruction text/columns.
- Doxygen headers updated per AGENTS.md: file-level headers in both include/tactical/FBattleDisplay.h and src/tactical/FBattleDisplay.cpp (Last Modified Jul 03, 2026, TMFR-03, author list extended with claude-sonnet-5 (medium)); full doc blocks added/updated for drawMoveShip(), buildMovePromptText(), and the new measureWrappedActionPromptWidth().
