Suggested behavioral cases (assert observed runtime outcomes, not source text):
1. writeU32 -> readU32 round-trip for several values (0, 1, 0xFF, 0x100,
   0x12345678, 0xFFFFFFFF) returns 0 and reproduces the value.
2. Little-endian byte layout: write a known value (e.g. 0x12345678) to a
   stringstream and assert the raw 4 bytes are 0x78,0x56,0x34,0x12 in order.
3. writeString/readString round-trip for a normal string and for a string with
   an embedded '\0' (e.g. std::string("ab\0cd", 5)) — assert the reloaded string
   equals the original AND has size()==5 (proves NUL is not truncated).
4. Empty-string round-trip: writeString("") then readString yields size()==0 and
   returns 0.
5. 64 KB cap rejection with no allocation: hand-craft a stream whose first 4
   bytes encode a length > kMaxSerializedStringBytes (e.g. 0x00020000 = 131072)
   with NO payload following, call readString, and assert it returns nonzero and
   does not attempt to read/allocate the oversized buffer (the stream should not
   be driven into reading 128 KB; a short buffer that would fail if it tried to
   read proves the no-allocation/early-reject path). Boundary check: a length of
   exactly 65536 is allowed (not rejected by the cap), 65537 is rejected.
6. writeString oversize guard: attempt to write a string longer than
   kMaxSerializedStringBytes and assert nonzero return (note: allocating a
   >64 KB std::string in the test is acceptable here since it is test-owned).
7. Failed-stream returns: put a stringstream into failbit state (or an empty
   input stream at EOF) and assert write/read/writeU32/readU32/writeString/
   readString each return nonzero.
8. Constants: assert kMaxSerializedStringBytes==65536, kSaveFormatVersion==1,
   and that writing kSaveMagic via writeU32 emits bytes 'S','S','W','1'
   (0x53,0x53,0x57,0x31). (Supplemental to the behavioral cases above.)
