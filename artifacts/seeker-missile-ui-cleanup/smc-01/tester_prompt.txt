## API Specifications

### 1. `bool deactivateActiveSeekerByID(unsigned int seekerID)`

**Purpose**: Deactivate an active seeker owned by the moving player.

**Scope**: Works only on seeker missiles that are:
- Owned by the current moving player
- Currently in active state
- Match the requested seekerID

**Return Semantics**:
- Returns **true** when an active, moving-player-owned seeker with the specified ID was found and successfully transitioned to inactive state
- Returns **false** when:
  - No seeker with that ID exists
  - The seeker exists but is not active
  - The seeker exists but is not owned by the moving player

**Expected Inverse Relationship**: This method is the exact inverse of `activateSelectedInactiveSeeker`. If you activate a seeker and then deactivate it by ID, it should return to its original inactive state.

### 2. `bool activateInactiveSeekerAtHex(const FPoint& hex)`

**Purpose**: Activate exactly one inactive seeker located at a given hex.

**Deterministic Selection**: When multiple inactive seekers share the same hex, the method **deterministically selects the lowest seekerID**. This ensures:
- Consistent behavior across test runs
- Predictable seeker activation order
- No random selection issues

**Return Semantics**:
- Returns **true** when an inactive, moving-player-owned seeker at the specified hex was found and successfully transitioned to active state
- Returns **false** when:
  - No seeker exists at the hex
  - All seekers at the hex are already active
  - All seekers at the hex are owned by the opposing player
  - The hex is out of bounds

**Side Effect**: Activates exactly one seeker per call. Multiple calls to the same hex with multiple stacked seekers will activate them one at a time.

### 3. `std::vector<FTacticalSeekerMissileState> getActiveSeekersByMovingPlayer()`

**Purpose**: Return all active seekers owned by the moving player for UI listing and navigation.

**Scope**: Returns only seekers that are:
- Owned by the current moving player
- Currently in active state
- From any hex on the battlefield

**Return Value**: A vector of FTacticalSeekerMissileState records containing the complete seeker state including:
- seekerID
- ownerID
- hex location
- heading
- active status (always true in returned results)
- movementTurn
- movementAllowance
- source tracking info

**Side Effects**: No side effects; this is a read-only accessor.

   - Subsequent activations at the same hex select the next-lowest ID

3. **getActiveSeekersByMovingPlayer**:

   - Returns only active seekers (not inactive ones)
   - Returns only moving player's seekers (not enemy seekers)
   - Includes seekers from all hexes on the battlefield
   - Returned vector is a copy (modifications don't affect model state)
