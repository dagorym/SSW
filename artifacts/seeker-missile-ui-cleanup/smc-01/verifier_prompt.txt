Your role is 'verifier'. Your task is as follows:

Verify the SMC-01 (Seeker Missile UI Cleanup — Stage 1) implementation and documentation.

**Story Context**

SMC-01 adds three new model-side APIs to support the reworked seeker activation screen UX:
1. `deactivateActiveSeekerByID()` — deactivate a single active seeker by ID (active → inactive) for the moving player
2. `activateInactiveSeekerAtHex()` — activate exactly one inactive seeker at a given hex for the moving player (deterministic when multiple share the hex)
3. `getActiveSeekersByMovingPlayer()` — const accessor returning all currently-active seekers owned by the moving player

Both FTacticalGame (model) and FBattleScreen (screen) receive these APIs. FBattleScreen methods are pass-through delegations that trigger a redraw on state change.

- `include/tactical/FTacticalGame.h` — three new method declarations with complete Doxygen documentation
- `src/tactical/FTacticalGame.cpp` — three method implementations (deterministic seeker selection, state transitions, filtered query)
- `include/tactical/FBattleScreen.h` — three pass-through delegation declarations with delegation-aware Doxygen documentation
- `src/tactical/FBattleScreen.cpp` — three delegation implementations (model call + redraw logic)

**Acceptance Criteria**

1. **Model Behavior (FTacticalGame)**
   - `deactivateActiveSeekerByID(seekerID)`:
     - Returns true when an active, moving-player-owned seeker with that ID was found and transitioned to inactive
     - Returns false if the seeker doesn't exist, is already inactive, or is owned by the opponent
     - Inactive and opponent-owned seekers are unaffected
   - `activateInactiveSeekerAtHex(hex)`:
     - Returns true when an inactive, moving-player-owned seeker at that hex was found and activated
     - Returns false if no such seeker exists or the hex is out of bounds
     - When multiple inactive seekers share the hex, selects deterministically (lowest seekerID)
     - Behavior is independent of the current activation phase (callable anytime)
   - `getActiveSeekersByMovingPlayer()`:
     - Returns a copy of all active seeker records owned by the moving player
     - Includes all active seekers across all hexes (not just the selected hex)
     - Suitable for UI listing (no wx types; returns pure tactical state)

2. **Screen Delegation (FBattleScreen)**
   - `deactivateActiveSeekerByID(seekerID)`:
     - Delegates to `m_tacticalGame->deactivateActiveSeekerByID(seekerID)`
     - Calls `reDraw()` only if the model method returned true (state changed)
     - Returns the model method's result
   - `activateInactiveSeekerAtHex(hex)`:
     - Delegates to `m_tacticalGame->activateInactiveSeekerAtHex(hex)`
     - Calls `reDraw()` only if the model method returned true (state changed)
     - Returns the model method's result
   - `getActiveSeekersByMovingPlayer()`:
     - Delegates to `m_tacticalGame->getActiveSeekersByMovingPlayer()`
     - Returns the model's result unchanged

3. **No wx Headers in Model**
   - `include/tactical/FTacticalGame.h` contains no active wx includes (only tactical types)
   - Seeker state is returned as `std::vector<FTacticalSeekerMissileState>` (no wx wrappers)

4. **Documentation**
