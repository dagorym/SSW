## Summary

Three new seeker missile APIs have been implemented and tested in FTacticalGame. This prompt provides the information needed to document these APIs.

## Implementation Details

### API 1: deactivateActiveSeekerByID()

**Location:**
- Header: `include/tactical/FTacticalGame.h` (around line 280)
- Implementation: `src/tactical/FTacticalGame.cpp`

**Signature:**
```cpp
void deactivateActiveSeekerByID(int seekerId);
```

**Purpose:**
Deactivates an active seeker missile by its ID. Removes the seeker from the active seekers tracking container and returns it to inactive state. This API allows the tactical UI to deactivate seekers that have reached their destination, been destroyed, or need to be removed from play for game-rule reasons.

**Parameters:**
- `seekerId` (int): The unique identifier of the active seeker to deactivate.

**Behavior:**
- If the seeker ID exists in active seekers, it is removed from active tracking
- If the seeker ID does not exist, the call is a graceful no-op with no side effects
- No exceptions are thrown for non-existent IDs

**Integration Points:**
- Works with movement phase seeker detection in `checkForActiveSeekersOnPath()`
- Compatible with seeker damage application via `applyMovementSeekerDamage()`
- Used to manage active seeker state between tactical phases

### API 2: activateInactiveSeekerAtHex()

**Location:**
- Header: `include/tactical/FTacticalGame.h` (around line 282)
- Implementation: `src/tactical/FTacticalGame.cpp`

**Signature:**
```cpp
void activateInactiveSeekerAtHex(int seekerId, const FPoint& hexPos);
```

**Purpose:**
Activates an inactive seeker missile and positions it at a specific hex on the tactical board. This API allows the tactical UI to bring seekers into active play by specifying their starting position. Activated seekers will be detected during subsequent movement collision checks.

**Parameters:**
- `seekerId` (int): The unique identifier of the inactive seeker to activate.
- `hexPos` (const FPoint&): The hex position where the seeker should be placed. Uses FPoint coordinates consistent with the FTacticalGame hex map.

**Behavior:**
- If the seeker ID exists in inactive seekers, it is moved to active tracking at the specified position
- The seeker's hex position is updated to the provided coordinates
- If the seeker ID does not exist or is already active, the call is a graceful no-op with no side effects
- No exceptions are thrown for edge cases

**Integration Points:**
- Positions seekers in the tactical hex map coordinate system
- Integrated with hex occupancy and collision detection
- Works with movement phase seeker detection via `checkForActiveSeekersOnPath()`
- Compatible with mine and occupancy tracking

### API 3: getActiveSeekersByMovingPlayer()

**Location:**
- Header: `include/tactical/FTacticalGame.h` (around line 284)
- Implementation: `src/tactical/FTacticalGame.cpp`

**Signature:**
```cpp
std::vector<int> getActiveSeekersByMovingPlayer(int playerId) const;
```

**Purpose:**
Returns a vector of all active seeker IDs currently owned by a specific player. This API allows the tactical UI to query which seekers a particular player has in active play, enabling UI rendering, seeker control panels, and game-rule enforcement.

**Parameters:**
- `playerId` (int): The unique identifier of the player whose active seekers should be retrieved.

**Return Value:**
- `std::vector<int>`: A vector of seeker IDs owned by the specified player and currently in active state. Returns an empty vector if the player has no active seekers.

**Behavior:**
- Scans active seekers container and filters by player ownership
- Returns only seekers with matching player ID
- Returns empty vector if no active seekers exist for the player (not null, not error)
- Provides a snapshot of seeker state; subsequent modifications do not affect the returned vector
- Const method; does not modify model state

**Integration Points:**
- Enables UI iteration and control of player-owned seekers
- Used to query seeker state for rendering and UI feedback
- Compatible with deactivateActiveSeekerByID() for controlling seekers from the returned list
