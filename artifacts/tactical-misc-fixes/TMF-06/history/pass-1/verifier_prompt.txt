## Story summary

TMF-06 added a `weaponsFired` count to `FTacticalCombatReportSummary` and guarded `showTacticalDamageSummaryDialog` in both fire-done handlers so that a fire phase in which no weapon actually fired skips the damage-summary dialog entirely. The mine-damage path calls `ITacticalUI::showDamageSummary` directly and is unaffected.

## Acceptance criteria

- **AC1**: `FTacticalCombatReportSummary.weaponsFired` is set by `buildTacticalCombatReportSummary` to `(int)report.attacks.size()` — zero for an empty report, 2 for a report with two attack entries.
- **AC2**: When `weaponsFired == 0`, `onDefensiveFireDone` and `onOffensiveFireDone` skip `showTacticalDamageSummaryDialog`.
- **AC3**: When `weaponsFired > 0` but all attacks missed/were intercepted (producing no damage rows, `ships` empty), the dialog is still shown — `weaponsFired == 1` with `ships.empty() == true` satisfies the guard.
- **AC4**: No regression — 249 TacticalTests pass; mine-damage report path unaffected.
